$ make --how

Cách nó chạy

  yêu cầu → INTAKE (hỏi có cấu trúc) → Feature Spec (Ready)
         → BUILD (TDD: test trước) → VERIFY (test xanh) → DONE
  • User chỉ làm 2 việc: cung cấp sự thật domain + duyệt cổng
  • Harness lo phần lặp lại: hỏi đúng thứ tự · soạn nháp · check gate · viết test

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 1 · INTAKE

LÀM GÌ: yêu cầu thô → Spec chuẩn (DoR: User Story + Gherkin AC)

TẠI SAO: nhốt hỗn loạn về 1 chuẩn TRƯỚC khi code

BỎ ĐI: code theo yêu cầu mơ hồ → build sai

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 2 · DECOMPOSE (cấp app)

LÀM GÌ: App Spec → nhiều Feature Spec nhỏ độc lập

TẠI SAO: chia nhỏ để kiểm được, chạy song song

BỎ ĐI: build cả app 1 cục → khó test, dễ ngợp

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 3 · BUILD (μ-build = TDD)

LÀM GÌ: mỗi AC → test TRƯỚC (RED) → code (GREEN) → refactor

TẠI SAO: code có test, từng tiêu chí kiểm được

BỎ ĐI: code không test = không tin được

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 4 · VERIFY

LÀM GÌ: test xanh + chạy thật → đối chiếu DoD

TẠI SAO: "done" cần bằng chứng, không tin lời

BỎ ĐI: "tưởng xong" mà hỏng → phát hiện muộn
