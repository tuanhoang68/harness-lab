$ make --how

Cách harness chạy

  yêu cầu → INTAKE → Feature Spec (Ready) → BUILD (TDD) → VERIFY → DONE
  • User chỉ: cung cấp domain + duyệt cổng
  • Harness lo: hỏi · soạn nháp · check gate · viết test

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 1 · INTAKE

  yêu cầu thô  ──▶  [ INTAKE ]  ──▶  Spec chuẩn ✓ DoR

→ nhốt hỗn loạn về 1 chuẩn TRƯỚC khi code

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 2 · DECOMPOSE

  App Spec  ──▶  [ DECOMPOSE ]  ──▶  Feature 1 · 2 · 3

→ chia nhỏ để kiểm được, chạy song song

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 3 · BUILD

  Feature Spec  ──▶  [ BUILD · test→code ]  ──▶  code + test ✓

→ TDD: test trước (RED) → code (GREEN)

INTAKE─▸DECOMPOSE─▸BUILD─▸VERIFY

Bước 4 · VERIFY

  code  ──▶  [ VERIFY · test✓ + chạy thật ]  ──▶  DONE ✓

→ tin bằng chứng, không tin lời
